**What**: The `version` field in `.pjt.lock` is higher than the version this projitect knows how to read. Means a teammate (or your future self) ran a newer projitect and committed the upgraded lockfile.

**How to fix**: Upgrade the `projitect` devDep to the version that wrote the lockfile (or newer). Run `pnpm add -D projitect@latest` (or your PM equivalent), then `pnpm pjt remodel`. Downgrading the lockfile by hand is not supported — the new fields would silently disappear.
