## Why projitect?

<CardGrid>
  <Card title="Composable blueprints" icon="puzzle">
    Drop blueprints into one file. Powered by **Effect**, so you get conditional logic, generator
    syntax, and typed errors for free — no shell scripts, no DSL.
  </Card>
  <Card title="Drift detection on every CI run" icon="approve-check">
    `pjt inspect` exits nonzero when your repo diverges from spec. No more *"works on my machine"*
    config rot. Add one line to your workflow and CI guarantees integrity.
  </Card>
  <Card title="Swap frameworks cleanly" icon="random">
    Remove a blueprint, the configuration it added is flagged. No orphaned `tsconfig` settings or
    stale `package.json` scripts when you migrate. Your repo stays as slim as your stack.
  </Card>
</CardGrid>

## What it looks like

Your project's blueprint file is one file at the root, called `.pjt.ts`:

```ts
import { pjt } from "projitect/cli"
import { gitignores } from "@projitect/blueprint-gitignore"
import { vitest } from "@projitect/blueprint-vitest"

export default pjt({
  blueprints: [
    gitignores.macOs(),
    gitignores.node(),
    gitignores.vscode(),
    vitest(), // vitest.config.ts + scripts + devDeps + coverage/ in .gitignore
  ],
})
```

That's the **whole spec**. Three commands act on it:

```bash
pjt init               # bootstrap projitect in this project
pjt remodel            # apply the blueprints to disk (non-destructive)
pjt inspect            # report drift (exit 1 in CI if the repo is out of spec)
pjt add <package>      # install a blueprint package and splice it into .pjt.ts
pjt build --force      # wipe and rebuild from scratch (requires clean git)
pjt explain <id>       # explain an error id
```

## See drift in action

After `pjt remodel`, your `.gitignore` has comment-fenced regions owned by each blueprint:

```text
# pjt:gitignore:macos start
.DS_Store
.AppleDouble
# ... rest of the macOS section ...
# pjt:gitignore:macos end
# pjt:gitignore:node start
node_modules/
.env
# ... rest of the Node section ...
# pjt:gitignore:node end
```

Now suppose a teammate hand-edits the file — say, they delete `.DS_Store`. Running
`pjt inspect` in CI catches it immediately:

```text
~ modify .gitignore — region pjt:gitignore:macos content drift
```

Exit code: `1`. PR fails. Run `pjt remodel` to fix.

## Status

**v0** ships region-mode end-to-end with the gitignore blueprint set, plus `pjt init`,
`pjt remodel`, `pjt inspect`, and `pjt explain`. Effect v4 (beta) under the hood.

The full execution model — merge mode for `package.json`, owned/seed modes, framework blueprints
(`react`, `next`, `tsconfig`, `eslint`), the worker-process sandbox, and the interactive
in-browser demo — is on the [roadmap](/docs/roadmap/).
