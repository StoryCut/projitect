---
title: pjt inspect
description: Report drift between the project and its blueprints. Nonzero exit on drift.
---

```bash
pjt inspect [--json]
```

Compares the project against `.pjt.ts`. Exits **0** if in sync, **1** if drift is detected.
Built for CI.

## Output

```text
~ modify .gitignore — region pjt:gitignore:macos content drift
+ create src/routes.generated.ts
  ok package.json
```

Three statuses per file:

| Symbol  | Meaning                                                                     |
|---------|-----------------------------------------------------------------------------|
| `+`     | File missing; will be created by `pjt remodel`.                             |
| `~`     | File exists but content drifts; will be updated by `pjt remodel`.           |
| (blank) | In sync.                                                                    |

## Flags

| Flag     | Description                                            |
|----------|--------------------------------------------------------|
| `--json` | Emit machine-readable JSON output. (v0.1)              |

## Use in CI

```yaml
- run: pnpm pjt inspect
```

Done. The job fails if any blueprint-managed file drifts.

## What it doesn't check

- Files no blueprint claims (untouched and uncomplained-about).
- `seed`-mode files after first write.
- Content outside region markers in `region`-mode files.

This is intentional. projitect manages what it owns; you own everything else.
