---
title: pjt build
description: Wipe the project tree and rebuild from blueprints. Destructive.
---

```bash
pjt build --force [--force-dirty] [--yes]
```

**Destructive.** Deletes every file under the project root except a small preserved set (see below), then runs `pjt remodel` to rebuild from the current blueprint plan. Intended for fresh greenfield scaffolds and one-shot migrations off another generator. **Not for day-to-day use** — that's what `pjt remodel` is for.

## Required + optional flags

| Flag             | Description                                                                                                       |
|------------------|-------------------------------------------------------------------------------------------------------------------|
| `--force`        | **Required.** Without it, `pjt build` refuses to run. The wipe is too destructive to fire by accident.            |
| `--force-dirty`  | Allow wiping even when `git status --porcelain` shows uncommitted changes. Default is to refuse on a dirty tree.  |
| `--yes`          | Skip the interactive confirmation prompt. Use this in scripts; otherwise the human at the keyboard confirms.      |

## What gets preserved

Everything else under the project root is deleted. These survive:

- `.git/`
- `node_modules/`
- `.pjt.ts`
- `.pjt.lock`
- `package.json`
- `pnpm-lock.yaml` / `package-lock.json` / `yarn.lock` / `bun.lockb`

The rationale: keep the manifest, the blueprint spec, the lockfile, the dependency store, and version control. Everything else regenerates from the plan.

## Prerequisites

| Check                                | Failure code                  |
|--------------------------------------|-------------------------------|
| Project root has `.git/`             | `pjt.git.not-a-repo`          |
| `git status --porcelain` is empty    | `pjt.apply.dirty-git` (override with `--force-dirty`) |
| `--force` flag is present            | `pjt.apply.dirty-git` (with a clarifying message)     |

## Output

```text
Wiped 4 entries:
  src
  tests
  docs
  scratch.md
Rebuilt 1 file from the plan:
  .gitignore
```

Or, when the user declines the confirmation:

```text
Cancelled. No changes made.
```

## When to use this

- Fresh project: `pjt init` then optionally `pjt build --force --yes` to wipe any leftovers and run the plan from scratch.
- Migrating off another scaffolder: delete the other tool's lockfile, install projitect via `pjt init`, run `pjt build --force` to clear the old tree and let blueprints rebuild what they own.
- Recovering from heavy hand-editing: when `pjt remodel` would produce a confusing diff, wipe and rebuild.

For everything else, use `pjt remodel` — it's non-destructive and handles drift, removals, and upgrades incrementally.
