When the blueprint tree depends on what's already in the project — say, only adding the Next.js
gitignore section if Next is actually a dependency — switch from the array form to the Effect
form.

## Why this works

`pjt` accepts either:

- `Array<Blueprint | DirectoryBlueprint>` — static, evaluated at module load
- `Effect<Array<Blueprint | DirectoryBlueprint>, Error, BlueprintFileSystem>` — dynamic, run by
  the planner with the same permission-gated FS your blueprints receive

The Effect form lets you yield to anything Effect can do — `readPackageJson`, `Effect.if`,
`Effect.when`, `Effect.forEach`. Project maintainers rarely need this, but it's there when they
do.
